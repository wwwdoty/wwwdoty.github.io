C++面向对象程序设计视频学习笔记

Part I Object Oriented Programming, OOP

1. 内联函数 inine

  1. 用它替代C中表达式形式的宏定义,解决一些频繁调用的小函数大量消耗空间（栈内存）的问题。
  2. 在类body内定义的的函数，自动成为内联函数候选人。
  3. 在类定义的函数想成为内联函数候选人，用inline关键字定义。
  4. 最终是否成为内联函数由编译器决定，一般函数体内代码较简单的会
    成为内联函数。

2. 构造函数 constructor(ctor)

  1. 函数名同class名，不需要返回类型。
  2. 特殊的初始化方式，使用:， 比在函数体内赋值快。
  3. 构造函数可以有多个，即重载(overloading),但注意内容不要冲突。
  4. 构造函数放在private区中，不能像通常方法那样创建对象让外界
    调用构造函数。（设计模式——Singleton)

3. 成员函数 member functions

在类中必须声明成员函数的函数原型，即返回类型，函数名，形参表。 函数体可以在类中或类外定义。 类名::函数名

  1. 常量成员函数
    如果成员函数中没有改变数据，建议在声明后函数体前加入const关键字，如果不加那么编译器会认为此函数可能会修改类的成员变量。
    这时如果再用const创建对象则会编译失败。
  2. 所有成员函数都有一个隐藏参数，this指针。谁调用了函数，就是默认
    this指针指向谁。

4. 参数传递 by reference / by value

  1. 函数参数传递，如果不需要改变参数值，建议使用const reference
    减小开销。
  2. 函数的返回值传递，一般也建议用reference。但当返回值是在函数内定义
    的局部变量，就不能返回引用。因为以但函数调用结束，该局部变量会被销毁。
    对于临时变量，也不能返回引用。
    • 临时对象用“typename ()”创建，

5. 友元 friend

友元机制允许特定的非成员函数访问一个类的私有成员，用关键字friend声明，只能出现在类定义的内部。

  1. 相同类的各个实例对象互为友元，可以通过彼此的内部方法调用传
    入参数的内部私有成员变量。

6. 操作符重载

7. 拷贝构造和拷贝赋值函数

带有指针成员的类，一定要自己写拷贝构造和拷贝赋值函数，否则只是浅拷贝，
会发生内存泄露。

  1. 拷贝赋值过程
    先清空左边待被赋值对象，再创造出和右边一样大小的内存，再把右边赋值
    给左边。
  2. 在重载“=”赋值运算符时需要检查自我赋值。如果不检查，在左右指向的
    是同一对象的情况下赋值，第一步先删掉左边待被赋值对象，其实把右边
    也删掉了，再企图访问右边，会产生不确定行为。

8. 堆栈及对象生命期

  1. stack栈是存在于某作用域scope的一块内存空间memory space。例如
    当调用函数，函数本身会形成一个stack来放置它接收的参数，及返回地址。
    在函数本体内声明的任何变量，其所使用的内存块都取自上述栈。

  2. 1 stack object的声明再作用域scope结束时结束，也被成为atuo object，
    因为它会被自动清理，及析构函数被调用。

  3. 2 static local object及static object，生命再作用域结束后仍存在，
    直到整个程序结束。

  4. 3 global object 生命也在整个程序结束后才结束。如c3变量（内存位置不清楚）

  5. heap堆是指由操作系统提供的一块global内存空间，程序可动态分配
    dynamic allocated从某中获得若干区块blocks。
    heap object，如下变量p, 生命在delete之际结束。如果不delete，
    作用域结束时，p所指的heap object对象仍存在，指针p的生命却已经结束。
    作用域之外再看不到p,也就无法delete。

9. 操作new和delete

  1. new 先分配内存再调用构造函数(隐含this参数)
  2. delete 先调用析构函数，删除动态分配的部分，再释放内存，
    删除指针。
  3. array new 一定要搭配 array delete，即 因为对于创建的对象是否是数组，编译器在分配内存时是不一样的，在数组
    元素前会插入记录元素个数的部分，这样编译器就会调用析构函数相应的次数
    如果只是用delete不加[]，只会调用一次析构函数删除对象第一个元素，剩下的
    没有处理。

10. static

  1. 可以通过static关键字定义静态成员变量，静态成员函数。
    成员函数通过this指针取到不同的成员变量，但静态成员函数没有this
    pointer,只能处理静态数据。
  2. 静态成员数据在类外要做一下定义。

11. 类模板 class template

template<typename T>
类中暂时不确定变量类型，可以在创建对象时再确认的变量类型可以
定义为T类型，创建对象时用classname object定义即可。
或者说，某个类里，某些变量的类型可以抽出来任意令用户指定。

12. 函数模板 function template

函数的返回对象类型，形参列表参数类型同类模板一样，定义的时候
在最开始加template<class T>,调用的时候正常使用不必指出实参
类型，编译器会进行参数推导。

13. 复合 composition 表示 has-a

表示有一个的关系 has a: container has a component
比如STL源码中queue的实现是通过定义queue c，利用c的相关操作函数完成, 这是两个类的一种复合关系。queue可以认为是容器适配器（adapter）

  1. 复合关系下的构造和析构
    • 构造由内而外，container先调用复合的component的默认构造函数，再执行自己的
    • 析构由外而内，container先执行自己，然后调用component的默认析构函数

14. 委托 delegation 表示 composition by reference

设计模式handle/body (pImpl, pointer to implentation)
有一个指向某个类的指针

15. 继承 Inheritance 表示 is-a

子类derived 是一个(is a) 父类base，继承了父类的数据，有public继承，private继承，protected继承

  1. 继承关系下的构造和析构

    • 构造由内而外，derived的构造函数先调用父类base的构造函数，再执行自己的
    • 析构由外而内，derived的析构函数先执行自己，然后调用base的析构函数

  2. base class 的析构函数必须是虚函数，否则会出现undefined behavior

  3. 子类继承了父类函数的调用权，可以通过子类的对象调用父类的函数

  4. Inheritance+Composition关系下的构造和析构

    • 构造由内而外，derived的构造函数先调用父类base的构造函数， 然后调用component的构造函数，再执行自己的
    • 析构由外而内，derived的析构函数先执行自己，然后调用component的析构函数，再调用base的析构函数

  5. delegation + Inheritance，设计模式 Composite， observer， prototype

16. 虚函数 Inheritance with virtual functions

  1. non-virtual 函数：你不希望derived class 重新定义(override，复写) 它.
  2. virtual 函数：你希望derived class 重新定义它，且你（父类？）对它已有默认定义。
  3. pure virtual 函数：你希望derived class 一定要重新定义它，你对它沒有默认定义。
