C++面向对象

1. 内联函数 inine

  1. 用它替代C中表达式形式的宏定义,解决一些频繁调用的小函数大量消耗空间（栈内存）的问题。
  2. 在类body内定义的的函数，自动成为内联函数候选人。
  3. 在类定义的函数想成为内联函数候选人，用inline关键字定义。
  4. 最终是否成为内联函数由编译器决定，一般函数体内代码较简单的会
    成为内联函数。

2. 构造函数 constructor(ctor)

  1. 函数名同class名，不需要返回类型。
  2. 特殊的初始化方式，使用:， 比在函数体内赋值快。
  3. 构造函数可以有多个，即重载(overloading),但注意内容不要冲突。
  4. 构造函数放在private区中，不能像通常方法那样创建对象让外界
    调用构造函数。（设计模式——Singleton)

3. 成员函数 member functions

在类中必须声明成员函数的函数原型，即返回类型，函数名，形参表。 函数体可以在类中或类外定义。 类名::函数名

  1. 常量成员函数
    如果成员函数中没有改变数据，建议在声明后函数体前加入const关键字，如果不加那么编译器会认为此函数可能会修改类的成员变量。
    这时如果再用const创建对象则会编译失败。
  2. 所有成员函数都有一个隐藏参数，this指针。谁调用了函数，就是默认
    this指针指向谁。

4. 参数传递 by reference / by value

  1. 函数参数传递，如果不需要改变参数值，建议使用const reference
    减小开销。
  2. 函数的返回值传递，一般也建议用reference。但当返回值是在函数内定义
    的局部变量，就不能返回引用。因为以但函数调用结束，该局部变量会被销毁。
    对于临时变量，也不能返回引用。
    • 临时对象用“typename ()”创建，

5. 友元 friend

友元机制允许特定的非成员函数访问一个类的私有成员，用关键字friend声明，只能出现在类定义的内部。

  1. 相同类的各个实例对象互为友元，可以通过彼此的内部方法调用传
    入参数的内部私有成员变量。
